# Q2-LabSchedulingSystem 机房排课系统

一个基于 Qt 6 / C++17 的桌面应用程序，模拟高校机房排课的完整业务流程，实现课程排课信息化与数字化管理。

## 项目背景

在传统的机房排课流程中，教师需要综合考虑学生上课进度、排课次数、师生可用时间、机房容量与空闲状态等多重因素，前往机房管理办公室人工查阅并确定排课计划，随后填写排课任务书与机房排课计划表。本系统旨在将这一纸质办公流程数字化，提供直观的可视化界面与自动化的冲突检测机制。

## 功能特性

### 1. 浏览模式
- 查看各机房的排课情况，隐藏操作区域以提供更大的可视空间
- 点击已排课时间段，在状态栏显示排课码、课程名、教师名等详情

### 2. 排课模式
- **排课码管理**：输入课程名称、教师、学院、班级、总排课次数等信息，自动生成唯一排课码
- **交互式排课**：双击空闲时间段完成排课，右键取消排课
- **冲突检测**：自动检查时间段占用、排课次数用尽、同一时段跨教室重复排课等冲突
- **跨教室排课**：支持在不同校区、不同机房间灵活调度
- **断点续排**：未排满的课程可在后续会话中通过课程名+教师名定位，继续排课

### 3. 报表模式
- 按课程名与教师名查询，自动生成包含日期、星期、午别、机房等字段的排课报表
- 报表以独立对话框展示，便于查看与核对

### 4. 统计功能
- 一键生成全学期机房使用情况统计表
- 按学院、教师、班级、课程名分类展示使用次数

### 5. 教室管理
- 预置 13 间机房（北区 5 间、未来城 3 间、西区 5 间），支持动态新增自定义机房
- 可修改各机房容纳人数（30~120 人）

### 6. 数据持久化
- 通过二进制数据流（QDataStream）将所有课程信息、排课数据、报表信息、自定义教室序列化保存为 `.stream` 文件
- 支持完整的加载恢复，包括动态新增的教室与按钮

## 架构设计

### MVC 架构

```
┌─────────────────────────────────────────────────┐
│                    View 层                       │
│  edrTableView (自定义 QTableView 派生类)          │
│  - 捕捉左击/右击事件，发射自定义信号               │
│  chartDialog / statDialog / roomDialog           │
│  - 报表、统计、教室管理对话框                      │
├─────────────────────────────────────────────────┤
│                  Controller 层                    │
│  edrMainWindow (主窗口)                           │
│  - 业务逻辑：排课码生成、冲突检测、排课调度          │
│  - UI 交互：模式切换、状态栏更新、信号槽连接         │
│  - 数据持久化：序列化/反序列化                      │
├─────────────────────────────────────────────────┤
│                   Model 层                        │
│  QStandardItemModel × N (每间教室一个模型)          │
│  - 存储 20 周 × 每日多时段的排课数据                │
│  - 通过 item 的用户数据存储教室容量等元信息          │
└─────────────────────────────────────────────────┘
```

### 核心数据结构

| 类型 | 用途 |
|------|------|
| `CourseInfo` | 存储课程名称、教师、学院、班级、总/剩余排课次数 |
| `ChartInfo` | 存储单次排课的报表信息（日期、星期、午别、机房） |
| `QList<CourseInfo>` | 顺序存储所有已创建课程 |
| `QList<QMap<int,ChartInfo>*>` | 双重指针容器，每个课程对应一个 QMap，按时间键值存储报表记录 |
| `QList<QStandardItemModel*>` | 动态教室的模型指针容器 |
| `QList<QPushButton*>` | 动态教室的按钮指针容器 |

## 技术要点

- **Qt Model/View 架构**：每间教室对应独立的 QStandardItemModel，通过切换 TableView 的模型实现教室间的无缝切换
- **自定义事件派发**：继承 QTableView 为 edrTableView，重写鼠标事件以区分左键（查看信息）和右键（取消排课）操作
- **双重指针容器**：使用 `QList<QMap<int,ChartInfo>*>` 实现多课程报表数据的灵活管理，每个课程拥有独立的报表映射
- **二进制序列化**：通过 QDataStream 实现自定义数据类型的完整序列化与反序列化，涵盖课程信息、报表数据、模型状态、动态教室配置
- **排课冲突检测**：多维度校验——时间段占用、排课次数、同一时段跨教室重复排课、表头保护
- **动态 UI 构建**：运行时创建按钮、模型、信号槽连接，支持用户自定义教室的完整生命周期管理
- **单元格合并与颜色编码**：通过 QTableView 的 span 机制合并表头，使用不同背景色区分表头、日期、时段、已排课项目

## 开发收获

本项目是独立完成的第一个完整 Qt 桌面应用，过程中的主要收获：

- **系统性学习 Qt 框架**：通读《Qt 6 C++ 开发指南》，结合官方文档（QGraphicsScene、QStandardItemModel、Graphics View Framework 等），建立对 Qt 组件体系的完整认知
- **MVC 思维的实践**：将 View（显示）、Model（数据载体）、Item（数据单元）三者的关系理清后，编码思路显著顺畅
- **C++ 进阶特性的实战**：首次在实际项目中运用类型向下转换（downcast）和双重指针结构
- **工程化意识**：认识到命名规范化、代码结构优化的重要性，为后续项目积累了宝贵经验

## 环境要求

- **Qt** >= 6.0（使用了 core、gui、widgets 模块）
- **C++ 标准**：C++17
- **编译器**：MinGW 64-bit 或 MSVC（推荐 MinGW）
- **构建工具**：qmake

## 编译步骤

### 方式一：命令行编译（qmake）

```bash
# 1. 确保 Qt 和 MinGW 已添加到 PATH 环境变量
# 例如：C:\Qt\6.7.2\mingw_64\bin 和 C:\Qt\Tools\mingw1120_64\bin

# 2. 创建构建目录
mkdir build
cd build

# 3. 运行 qmake 生成 Makefile
qmake ../Q2ClassArrSys.pro

# 4. 编译项目
mingw32-make        # Windows MinGW
# 或
make                # Linux/macOS
```

### 方式二：Qt Creator 编译

1. 打开 Qt Creator
2. 选择 **文件 > 打开文件或项目**
3. 选择 `Q2ClassArrSys.pro` 文件
4. 配置构建套件（Kit），选择 Qt 6.x + MinGW 64-bit
5. 点击左下角的 **运行** 按钮（绿色三角）即可编译并运行
